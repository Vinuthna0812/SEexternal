# ✅ 1) Ubuntu EC2 – Connect using .pem (SSH from Local)

### 🔹 Create Instance

1. Login → AWS Academy Learner Lab → Start Lab
2. Click AWS → Search EC2
3. Click Launch Instance
4. Enter Name
5. In AMI → Select Ubuntu
6. In Key pair → Click Create new key pair
7. Select .pem format
8. Click Create key pair (file downloads)
9. Click Launch Instance

---

### 🔹 Connect using SSH

10. Select instance → Click Connect

11. Go to SSH Client tab

12. Copy ssh -i command

13. Open PowerShell / Git Bash

14. Go to key folder:

```
cd path_of_key
```

15. Paste ssh -i command → Enter

16. Verify:

```
whoami
```

Output: ubuntu

17. Exit:

```
exit
```

---

# ✅ 2) Ubuntu EC2 – Connect using PuTTY (.ppk directly)

### 🔹 Create Instance

1. Launch Instance
2. Select Ubuntu
3. Create key pair → Select .ppk format
4. Launch Instance

---

### 🔹 Configure Security Group

1. EC2 → Instances → Select instance
2. Click Security
3. Click Security Group
4. Edit inbound rules
5. Ensure SSH – Port 22 – Source My IP
6. Save

---

### 🔹 Connect using PuTTY

1. Open PuTTY
2. Paste Public IPv4 in Host Name
3. Go to Connection → SSH → Auth → Credentials
4. Browse → Select .ppk file
5. Click Open
6. Accept

Login as:

```
ubuntu
```

Verify:

```
whoami
```

# ✅ 1a) Create and Attach EBS Volume to EC2

### 🔹 Create EBS Volume

1. Login → AWS Console
2. Search **EC2**
3. Left panel → Click **Volumes**
4. Click **Create Volume**
5. Select:

   * Volume type (General Purpose SSD – gp2/gp3)
   * Size (e.g., 8 GiB)
   * Availability Zone → **Same as EC2 instance**
6. Click **Create Volume**

---

### 🔹 Attach Volume

7. Select created volume
8. Click **Actions → Attach Volume**
9. Select EC2 Instance
10. Device name → keep default (/dev/sdf)
11. Click **Attach**

---

# ✅ Mount EBS Volume in Linux EC2

12. Connect to EC2 via SSH
13. Check disks:

```
lsblk
```

14. Format volume:

```
sudo mkfs -t ext4 /dev/xvdf
```

15. Create directory:

```
sudo mkdir /data
```

16. Mount volume:

```
sudo mount /dev/xvdf /data
```

17. Verify:

```
df -h
```

---

# ✅ Make Mount Permanent (Data persists after reboot)

18. Get UUID:

sudo blkid
sudo nano /etc/fstab
UUID=your-uuid /data ext4 defaults,nofail 0 2
```
21. Save & exit
22. Test:

sudo mount -a

# ✅ 1b) Increase / Decrease CPU & RAM

⚠ You cannot directly change CPU/RAM.
You must change **Instance Type**.

### 🔹 Steps

1. EC2 → Instances
2. Select instance
3. Click **Instance State → Stop**
4. Wait until stopped
5. Click **Actions → Instance Settings → Change Instance Type**
6. Select new type (e.g., t2.micro → t2.medium)
7. Click **Apply**
8. Click **Instance State → Start**

CPU & RAM updated.

# ✅ PART 1: Create EC2 Instance

### Step 1: Launch Instance

1. Go to EC2 → Click **Launch Instance**
2. Choose **AMI**

   * Select Amazon Linux / Ubuntu (as required)
3. Choose **Instance Type**

   * Example: t2.micro

---

### Step 2: Configure Network

4. Select **VPC**
5. Select **Subnet** (Public or Private as needed)
6. Enable Auto-assign Public IP (if public access required)

---

### Step 3: Create / Configure Security Group

7. Create new Security Group
8. Add **Inbound Rules**:

| Type | Port | Source                     |
| ---- | ---- | -------------------------- |
| SSH  | 22   | My IP                      |
| NFS  | 2049 | EFS Security Group (later) |

(For mounting EFS, port 2049 must be open.)

9. Outbound: Allow All (default)

---

### Step 4: Create Key Pair

10. Select existing key pair OR create new key pair
11. Download `.pem` file

---

### Step 5: Launch Instance

12. Click **Launch**
13. Connect using SSH:

```
ssh -i key.pem ec2-user@public-ip
```

---

| ---- | ---- | ------------------ |

# ✅ A) Create S3 Bucket + Upload Object + Make Public

1. Go to AWS Console → Services → S3
2. Click **Create Bucket**
3. Enter Bucket Name: `sars-bucket-2026`
4. Select Region: `Asia Pacific (Mumbai)`
5. Under Block Public Access → Uncheck **Block all public access**
6. Confirm warning checkbox
7. Click **Create Bucket**
8. Open bucket `sars-bucket-2026`
9. Click **Upload** → Add file `index.html` → Click Upload
10. Select uploaded file → Go to **Permissions**
11. Click **Edit ACL**
12. Enable **Public Read** → Save changes

# ✅ B) Enable Versioning + Restore Object

1. Open bucket `sars-bucket-2026`
2. Go to **Properties** tab
3. Scroll to Versioning → Click **Edit**
4. Select **Enable** → Save
5. Upload new version of `index.html`
6. Delete `index.html`
7. Go to `index.html` → Click **Versions** tab
8. Select previous version
9. Click **Download** or **Restore**

7. Click **Replication Rules**
8. Click **Create Rule**
9. Enter Rule Name: `crr-rule`
10. Choose Destination Bucket: `sars-dest-2026`
11. Create new IAM Role -Labrole
12. Save Rule

# ✅ A) Create VPC for 3-Tier Architecture (Web–App–DB)

1. Go to VPC → Create VPC
2. Enter Name: `sars-vpc`
3. Enter CIDR Block: `10.0.0.0/16`
4. Click Create

---

### Create Subnets

5. Create Subnet → Name: `sars-public-1` → CIDR: `10.0.1.0/24` → AZ: ap-south-1a
6. Create Subnet → Name: `sars-app-1` → CIDR: `10.0.2.0/24` → AZ: ap-south-1a
7. Create Subnet → Name: `sars-db-1` → CIDR: `10.0.3.0/24` → AZ: ap-south-1a

---

### Create Internet Gateway

8. Create Internet Gateway → Name: `sars-igw`
9. Attach `sars-igw` to `sars-vpc`

---

### Configure Public Route Table

10. Create Route Table → Name: `sars-public-rt`
11. Add Route: Destination `0.0.0.0/0` → Target `sars-igw`
12. Associate `sars-public-1` subnet to this route table

---

### Create NAT Gateway (For Private Subnets)

13. Allocate Elastic IP
14. Create NAT Gateway in `sars-public-1` → Name: `sars-nat`
15. Create Private Route Table → Add route `0.0.0.0/0` → Target `sars-nat` → Associate with `sars-app-1` and `sars-db-1`

---

# ✅ B) Launch EC2 in Public Subnet


1. Go to EC2 → Launch Instance
2. Name: `sars-web`
3. Choose AMI (Amazon Linux)
4. Choose Instance Type: t2.micro
5. Select VPC: `sars-vpc`
6. Select Subnet: `sars-public-1`
7. Enable Auto-assign Public IP
8. Create Security Group `sars-web-sg`
9. Add Inbound Rule: HTTP (80) → Source: 0.0.0.0/0
10. Add Inbound Rule: SSH (22) → Source: Your Company IP

Launch instance.

# ✅ C) Create RDS in Private Subnet
